## 1. 产品定位

**产品名称：冷静购**

冷静购是一款帮助用户在冲动消费前冷静下来的理性消费助手。产品对标 Forest 的**习惯养成与成就反馈机制**，但核心场景从「专注学习」转为「冷静消费」：用户在想下单前，将商品截图或链接导入 App，进入冷静期，获得情绪安抚、购买必要性分析、真实口碑佐证、愿望池沉淀和成长反馈，从而逐渐形成「先冷静，再购买」的消费习惯。

产品不以说教、记账或强制拦截为核心，而是通过**情绪价值、养成体验和成就感**，让用户愿意反复使用。

- **产品形态**：移动端优先的 Web 应用，后续可扩展为小程序或 App。
- **MVP 品类**：全品类。
- **MVP 平台**：淘宝、拼多多、京东；不支持二手平台。
- **重点渠道**：全部。

## 2. 目标 / 目的

- 降低冲动购物。
- 对标 Forest 的不是「学习场景」，而是「养成机制」：计时、成长、连续使用、成就反馈。
- 产品体验强调陪伴感、情绪价值和正向反馈，而不是简单告诉用户「不要买」。
- 在用户首次使用时通过「消费性格测试」了解用户心理，并在后续使用中持续记住用户偏好，让用户感到产品越来越懂自己。

## 3. 目标用户

### 3.1 核心用户

- 经常在淘宝、拼多多等平台产生冲动消费的用户。
- 对低价、限时、满减、直播间氛围、种草内容敏感的用户。
- 想控制消费，但不喜欢传统记账 App 或强约束工具的用户。
- 希望通过轻量、可持续的方式改善消费习惯的学生和年轻上班族。

### 3.2 用户痛点

- 购物平台通过低价、倒计时、库存、补贴、满减等机制制造紧迫感。
- 用户下单时往往处在情绪驱动中，难以判断真实需求。
- 传统记账工具只能事后复盘，不能在购买前介入。
- 纯理性提醒容易让用户有压力，难以长期坚持。
- **信息差**：电商详情页只呈现「买它的理由」，用户很难在下单前看到真实使用反馈与避雷信息。
- 用户需要的是「被理解」和「被陪伴着冷静下来」，而不是被批评。

## 4. 产品目标

### 4.1 MVP 目标

- 支持用户导入商品截图。
- 支持识别淘宝、拼多多、京东商品链接或截图来源。
- 能对任意品类商品给出冷静消费分析。
- **用商品信息去小红书等平台抓真实口碑，作为分析佐证与冷静期定制问题的素材**（§5.4）。
- 首次使用时提供一份轻量消费性格测试，包装成有趣测试而不是答题问卷。
- 提供冷静期，帮助用户延迟决策（流程见 §5.6）。
- 提供沉淀 / 愿望池功能，让用户把想买的商品暂存起来。
- 通过成长反馈和成就体系提升用户复用意愿。

### 4.2 长期目标

- 帮助用户形成「购买前先冷静」的习惯。
- 建立用户个人消费偏好、冲动触发点和省钱成就档案。
- 随着用户不断使用，逐步记住用户基本信息、消费偏好、常见冲动场景和有效提醒方式，形成「懂我」的个性化体验。
- 从单次消费判断工具，成长为长期消费行为管理助手。

## 5. 核心功能

### 5.1 消费性格测试

用户首次进入产品时，不直接呈现严肃问卷，而是以「测测你的消费人格」「你的购物冲动雷达是什么类型」等轻松形式完成测试。

该功能的目标是了解用户购物心理，建立初始用户画像，为后续冷静分析、情绪安抚和成长反馈提供个性化基础。

> **设计要求（批注 #0）**：测试题目要**结合后续分析所需的信息一起设计**——每道题都应映射到画像里一个会真正影响分析的维度，避免「测了但用不上」。

> **双层数据设计（批注 #1）**：对用户**展示趣味化的人格标签**；系统侧实际记录**结构化数值**（雷达图各维度分值），仅用于影响后续分析，不向用户暴露评分细节。

测试内容包括：

- 常见冲动消费场景：深夜购物、直播间、低价秒杀、满减凑单、情绪低落、奖励自己、社交种草等。
- 高风险品类：服饰、美妆、食品、家居、数码、兴趣消费、日用品囤货等。
- 易心动价格带：用户最容易「不贵就买了」的价格区间。
- 决策风格：比价型、情绪型、囤货型、颜值型、奖励型、从众型等。
- 提醒偏好：温柔陪伴、理性分析、幽默吐槽、直接拦一下等。

测试结果输出：

- 一个有趣的消费人格标签（展示层）。系统侧形成「五维消费冲动雷达图」（维度范例：价格敏感度、情绪代偿度、社交种草率、囤货安全感、颜值即正义）。
- 2~3 个主要冲动触发点。
- 一条适合用户的冷静策略。

人格标签示例：奖励型购物者 / 低价心动型 / 种草易感型 / 囤货安全感型 / 颜值即正义型。

> 消费人格标签只用于趣味化表达和个性化体验，不作为严肃心理诊断。

### 5.2 用户记忆与个性化理解

产品需要在用户不断使用后逐步记住用户的基本信息、偏好和行为模式，让用户感到「这个 App 真的懂我」。

系统应持续沉淀以下信息：

- 常买品类和高风险品类。
- 常见价格区间和易冲动价格带。
- 冲动触发点：低价、限时、满减、直播、颜值、情绪补偿、朋友推荐等。
- 历史选择：哪些商品最终购买，哪些商品冷静后放弃。
- 有效提醒方式：用户更容易接受理性分析、情绪安抚、幽默提醒还是成就反馈。
- 长期目标：省钱、减少囤积、控制情绪消费、提升生活品质等。

> **批注 #2**：「常用购物平台」是否记录视其对分析是否有实际作用而定——若对判断帮助不大，则**不作为画像记录项**（识别时仍可临时获取来源，但不长期沉淀）。

个性化应用方式：

- 冷静分析中结合历史习惯，例如「你上次类似的小家电后来没有购买，这次也可以先放一放」。
- 情绪安抚文案匹配用户偏好，例如温柔陪伴型用户看到更柔和的表达。
- 愿望池提醒更贴合用户风险，例如低价心动型用户看到「小额也会累积」的提醒。
- 成就反馈更贴近用户目标，例如「这个月你已经守住了 3 次深夜冲动」。

用户记忆应保持克制和透明，避免让用户产生被监视感。

### 5.3 商品识别

用户可以通过以下方式导入商品：

- **上传商品截图**。
  - 范围降级：系统通过 OCR 仅**优先抓取屏幕中的「价格」与「营销关键词」**（如：限时、秒杀、满减）；若商品品类和名称识别置信度低，则弹窗由用户一键勾选大类（服饰 / 美妆 / 数码 / 食品 / 其他）。
- 粘贴淘宝商品链接。
- 粘贴拼多多商品链接。
- 粘贴京东商品链接。

MVP 阶段识别内容包括：

- 商品名称。
- 商品价格。
- 商品来源平台（注：是否长期记录见 §5.2 / 批注 #2；识别平台来源是否必做见批注 #6）。
- 商品品类。
- 页面中的促销刺激信息，如限时、补贴、低库存、满减、券后价等。

### 5.4 真实口碑佐证（新增 · 信息差填平）

> 这是冷静购区别于「记账 / 拦截类」工具的**差异化能力**，对应用户痛点中的「信息差」。
>
> ✅ **取数方式已采纳方案 B（分层）**：用户提供 + AI 先验为主，**搜索 API 为稳定真实来源**，免费抓取（DDG/小红书笔记）作 best-effort 增强，逐层优雅降级。已落地见 `backend/app/services/insight`（`python app/services/insight/demo.py 破壁机` 可跑）。
> 下方 v0.2 正文中「直接抓小红书检索笔记/评论」为旧设想，已被方案 B 取代，以提案 + 代码为准；本节正文待下次统一回填。

识别出商品后（§5.3），系统用商品名称 / 品类关键词去**小红书等平台**检索真实笔记与评论，再用 AI 提炼出对「该不该买」最有价值的信号：

- 避雷 / 后悔点（「买了后悔」「踩坑」）。
- 闲置信号（「买了吃灰」「用了一次就闲置」）。
- 智商税 / 性价比口碑。
- 常见替代品建议。
- 真实使用体验与适用人群。

**两个用途：**

1. **反哺冷静分析报告（§5.5）**：作为「真实世界佐证」，让分析有据可依，而非泛泛而谈。
2. **生成冷静期定制化问题（§5.6）**：把佐证变成一个直击要害的问题，例如：
   > 「小红书上不少人提到这款破壁机买完基本闲置，你确定自己会经常用吗？」

**工程约束（必须遵守）：**

- **异步 + 缓存**：抓取慢且有反爬风险，按商品 / 品类缓存结果，避免每次实时全量抓取。
- **优雅降级**：抓取失败 / 无数据时，回退为纯 LLM 分析，不阻塞主流程。
- **去噪 / 去广告**：抓回的内容先过滤推广笔记与噪声，再交给 AI 提炼。
- **合规**：遵守目标平台的频率限制与 robots，仅在授权范围内采集；工具实现与已知问题见 [tools/xiaohongshu-scraper.md](tools/xiaohongshu-scraper.md)。

> MVP 阶段可先支持小红书一个来源；抖音 / 其它平台留待后续（与批注 §11「是否支持小红书、抖音种草场景」呼应——本能力即对该问题给出「是，先做小红书」的回答）。

### 5.5 冷静消费分析

系统基于**商品信息（§5.3）+ 真实口碑佐证（§5.4）+ 用户画像（§5.1/§5.2）**，生成轻量分析：

- 冲动风险等级。
- 当前购买动机判断。
- 是否可能存在替代品（可引用 §5.4 的替代品口碑）。
- 是否属于低价高频冲动品。
- 是否建议进入冷静期。
- 与用户历史偏好或冲动触发点的关联。
- **真实口碑佐证摘要**（来自 §5.4，标注来源，注明为社区观点而非结论）。

分析语气应保持温和、陪伴式表达，避免责备用户。

示例语气：

> 你不是不能买，只是可以先把决定权拿回来。

### 5.6 冷静期

> **流程简化（批注 #3、#7、#8）**：不预设复杂的多档强规则，采用清晰的状态机。

**决策状态机：**

1. 看完冷静分析 → 用户选择：**① 购买　② 放弃　③ 加入冷静期**。
2. 若加入冷静期，冷静结束后 → 再次询问：**① 购买　② 放弃　③ 继续冷静**。

**冷静期时长（MVP 简化）：**

- **短冷静**：适用于低价冲动商品，含「3 分钟心理突围战」轻互动。
  - 用户不需要死等，而是由 AI 陪他完成 3 个**基于消费人格的轻量互动**（例如：给冲动情绪打分、查看本月已省金额、深呼吸 5 秒，或回答一个**由 §5.4 口碑佐证生成的定制化问题**）。答完即可视为「冷静成功」，提前解锁成长值并打破电商的情绪劫持。
- **长冷静**：适用于中高价 / 非刚需商品（如 24 小时 / 用户自定义）。

> 时长是固定档位还是按价格 / 风险动态调整，列入 §11 待确认（批注 #3 的延伸）。

冷静期中，用户看到的是**成长反馈**，而不是单纯倒计时。

### 5.7 愿望池 / 沉淀池

用户可以将商品放入愿望池。

愿望池记录：

- 商品名称、商品价格、来源平台。
- 加入时间、冷静剩余时间。
- 当前状态：仍想买 / 已降温 / 已放弃 / 已购买。

愿望池的目标不是收藏商品，而是帮助用户观察欲望是否会自然降温。

### 5.8 养成与成就

对标 Forest 的成长机制，但**不照搬其视觉设定**（批注 #10）。

> **激励重心（批注 #4）**：成长机制**以激励「完成冷静过程」为主**（开始冷静、坚持到冷静结束、做出理性决定——这些用户可控、可诚实统计）；「守住 / 省下金额」作为**软性展示**，标注为「估算」，不作为硬性 KPI，避免诱导「为省钱而买更便宜的」。

机制：

- 每完成一次冷静期，获得成长值。
- 每放弃一次非必要购买，获得正向反馈。
- 连续使用形成 streak。
- **视觉隐喻待定（批注 #10）**：不直接沿用 Forest 的树 / 森林。候选方向——「心湖归于平静 / 水面波纹」「潮汐退去」「扑满 / 存钱罐积攒」「冷静值 / 理智账户进度」。最终包装另行讨论。

成就示例：

- 第一次冷静成功。
- 连续 7 天冷静消费。
- 本月守住 100 元（软性 / 估算）。
- 从愿望池放弃 5 件商品。
- 识别 3 个常见冲动触发点。

## 6. MVP 范围

### 6.1 必做

- 首次使用的消费性格测试。
- 基于测试结果生成消费人格标签和初始用户画像（展示标签 + 内部结构化数值）。
- 记录用户基础偏好、常见冲动触发点和有效提醒方式。
- 商品截图上传入口 + 淘宝 / 拼多多 / 京东 来源识别。
- 商品基础信息识别。
- **真实口碑佐证（小红书来源）+ 失败优雅降级**。
- 冲动风险提示与冷静消费分析。
- 冷静期（决策状态机 + 短冷静轻互动）。
- 愿望池。
- 成长值、连续天数、成就反馈。
- 温和情绪安抚文案。

### 6.2 暂不做

- 全平台自动抓取。
- 银行卡或支付账户连接。
- 社交排名。
- 复杂预算系统。
- 精准 OCR 后的全字段结构化识别。
- 复杂心理测评或医疗级心理判断。
- 用户画像的复杂账号体系和跨设备同步。

> **登录（批注 #5）**：手机号登录在 MVP 阶段**改为可选 / 暂缓**；MVP 用本地画像即可跑通（与 §11「画像存本地还是云端」一并确认）。

## 7. 关键用户流程

### 7.1 首次使用流程

1. 用户首次打开冷静购。
2. 系统以「消费性格测试」的形式引导用户完成 5~8 个轻量问题。（用户可选择跳过，后续弹窗再次弹出：「想解锁更懂你的专属 AI 守护人设？花 30 秒测测你的消费人格」）
3. 用户获得消费人格标签、主要冲动触发点和推荐冷静策略。
4. 系统将测试结果作为初始画像，用于后续商品分析、安抚文案和成就反馈。
5. 用户进入首页，开始使用截图识别、链接识别、冷静期和愿望池。

### 7.2 截图识别流程

1. 用户在淘宝 / 拼多多 / 京东看到想买的商品。
2. 用户截图并上传到冷静购。
3. 系统识别商品信息（平台来源识别为增强项，非阻塞——批注 #6）。
4. 系统抓取真实口碑佐证（§5.4），结合商品信息与用户画像生成冷静消费分析。
5. **用户选择：购买 / 放弃 / 加入冷静期**（批注 #7）。
6. 若加入冷静期，结束后用户重新决定：购买 / 放弃 / 继续冷静。

### 7.3 链接识别流程

1. 用户复制淘宝 / 拼多多 / 京东商品链接。
2. 用户粘贴到冷静购。
3. 系统识别来源平台和商品信息。
4. 系统抓取真实口碑佐证（§5.4），结合用户历史偏好判断冲动风险并生成分析。
5. **用户选择：购买 / 放弃 / 加入冷静期**（批注 #8）。

## 8. 体验原则

- 不说教：避免「你不应该买」。
- 不制造羞耻感：用户想买东西是正常情绪。
- 不只做工具：要让用户感到被陪伴。
- 正向反馈：强调「你守住了预算」「你拿回了选择权」。
- 轻量使用：从截图或链接进入，不增加复杂负担。
- 养成感强：每次冷静都应带来可感知的成长反馈。
- 懂用户但不冒犯：个性化表达要像朋友记得用户偏好，而不是像系统监控用户。
- 测试有趣：首次问卷应包装为性格测试，避免出现考试感和审问感。
- **佐证而非裁决**：口碑佐证呈现为「很多人这样说」，而不是「所以你不该买」，决定权始终在用户。

## 9. 隐私与信任原则

- 明确告知用户产品会记录偏好，用于提供更个性化的冷静建议。
- 用户应能查看、修改或清除自己的消费偏好和画像标签。
- 不将消费人格标签包装成专业心理诊断。
- 不用羞辱、恐吓、道德评判的方式解释用户行为。
- MVP 阶段优先记录与消费决策直接相关的信息，避免过度采集。
- 口碑佐证只展示聚合观点，不存储 / 不展示可定位到具体个人的敏感信息。

## 10. 成功指标

> **指标口径调整（批注 #9）**：区分「demo / MVP 阶段就能真实测量的」与「需要长期真实用户才有意义的」，避免设定一个月内无法验证的指标。

### 10.1 demo / MVP 阶段即可真实测量（建议作为比赛演示核心）

- 首次消费性格测试完成率。
- 截图 / 链接识别成功率。
- **口碑佐证命中率**（成功抓到可用真实评价的比例）。
- 冷静期启动率、完成率。
- 商品加入愿望池比例、愿望池最终放弃比例。
- **单次决策改变率**：进入分析 / 冷静后，用户由「想买」转为「放弃 / 加入冷静」的比例。

### 10.2 需长期 / 真实用户（北极星方向，不要求一个月内达成）

- 次日 / 7 日留存、连续使用天数。
- 用户月度冲动消费减少金额。
- 量化指标：「欲望降温 ROI」与「理智挽回总额」。
- 用户对「App 是否懂我」的主观评分。
- 用户是否形成购买前使用 App 的习惯。

## 11. 待确认问题

- 冷静期默认时长是固定档位还是按价格 / 风险等级动态调整（批注 #3 延伸；原稿提到的 20 分钟存疑）。
- 第一版截图识别是否只做 OCR + 规则判断，还是接入多模态识别。
- 愿望池是否需要商品降价提醒。
- 成长体系视觉隐喻最终方案（批注 #10，不照搬 Forest）。
- 口碑佐证除小红书外是否扩展抖音等种草场景（§5.4 已先做小红书）。
- 首次消费性格测试控制在多少题最合适，5 题、8 题还是允许跳过。
- 用户画像第一版存在本地还是云端账号体系；登录是否必做（批注 #5）。
- 个性化表达的边界如何把握，避免让用户觉得被过度分析。
- 养成机制以「完成冷静」还是「省钱数目」为主激励（批注 #4，本稿暂定以「完成冷静」为主）。

## 附录 B：真实口碑佐证设计细节

> ⚠️ 取数方式尚未拍板，以下为 v0.2 初始设想；候选方案与实测见 `docs/proposals/data-acquisition-strategy.md`，定稿后回填。

**触发时机**：§5.3 识别出商品之后、§5.5 生成分析之前（异步并行，超时则降级）。

**数据流：**

```
商品信息(名称/品类)
   └─▶ 检索小红书笔记/评论 (tools/ 爬虫，带缓存)
         └─▶ 去广告/去噪过滤
               └─▶ Claude 提炼结构化信号
                     ├─ 避雷/后悔点
                     ├─ 闲置("吃灰")信号
                     ├─ 智商税/性价比
                     ├─ 替代品建议
                     └─ 真实使用体验
                           ├─▶ §5.5 分析报告："真实世界佐证"
                           └─▶ §5.6 冷静期：生成定制化问题
```

**缓存键**：商品标准化名称 / 品类；命中缓存直接复用，避免重复抓取。

**降级策略**：抓取超时 / 无结果 / 被限流 → 跳过佐证，分析仅用商品信息 + 画像，前端不报错、不空白。

**实现映射**：抓取走 `backend/app/integrations/xhs_scraper`（封装 `tools/` 爬虫），提炼与编排走 `backend/app/services/insight`，分析消费走 `backend/app/services/analysis`。爬虫现状与修复见 [tools/xiaohongshu-scraper.md](tools/xiaohongshu-scraper.md)。
